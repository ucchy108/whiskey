# Work Logs

このディレクトリには、Claude Codeによる作業ログが記録されます。

## 目的

- 作業履歴の可視化
- チーム内での作業内容の共有
- 変更内容の追跡と振り返り
- ドキュメントとしての作業記録の蓄積

## ファイル命名規則

```
YYYY-MM-DD-<task-name>.md
```

**例**:
- `2025-12-25-storybook-setup.md`
- `2025-12-25-add-dashboard-header.md`
- `2025-12-26-fix-auth-bug.md`

## ログファイルの構成

各ログファイルには以下の情報が含まれます：

- **作業日**: 作業を実施した日付
- **ステータス**: 進行中 / 完了 / 保留
- **目的**: タスクの目的や背景
- **作業内容**: 時系列での作業詳細
- **変更ファイル**: 変更したファイルとその内容
- **課題・Todo**: 残タスクや今後の改善点
- **備考**: その他メモや注意事項

## 作業ログの利用方法

### 1. 現在の作業を確認

```bash
# 最新のログファイルを確認
ls -lt docs/work-logs/ | head -n 5
```

### 2. 特定のタスクを検索

```bash
# storybook関連の作業を検索
grep -l "storybook" docs/work-logs/*.md
```

### 3. 作業履歴の振り返り

各ログファイルを読んで、過去の作業内容や決定事項を確認できます。

## 自動記録

Claude Codeの`work-log` Skillにより、以下のタイミングで自動的に記録されます：

1. 新しいタスクや機能の実装開始時
2. 重要な変更の実施時
3. タスク完了時
