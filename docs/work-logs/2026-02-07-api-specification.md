# バックエンドAPI仕様書の作成

**作業日**: 2026-02-07
**担当**: Claude Code
**ステータス**: 完了

## 目的

バックエンドの全APIエンドポイントの仕様書を作成し、CLAUDE.mdから参照できるようにする。

## 作業内容

### 作業開始 - ハンドラー・ルーター調査

- 全ハンドラーファイルのリクエスト/レスポンス構造を確認
- ルーター設定からエンドポイント一覧を確認
- 認証要否の確認
- 認証ミドルウェア（Cookie/Session方式）を確認

### API仕様書の作成

- `docs/development/api-specification.md` を新規作成
- 全19エンドポイントの仕様を記載
- CLAUDE.mdの開発ガイドセクションにリンクを追加

## 完了サマリー

### 実装内容

- 全19エンドポイントのAPI仕様書を作成
  - ヘルスチェック: 1エンドポイント
  - ユーザーAPI: 5エンドポイント（登録、ログイン、ログアウト、情報取得、パスワード変更）
  - ワークアウトAPI: 8エンドポイント（記録、一覧、詳細、メモ更新、セット追加/削除、ワークアウト削除、コントリビューション）
  - エクササイズAPI: 5エンドポイント（CRUD + 一覧取得）
- 各エンドポイントにリクエスト/レスポンスの型定義、ステータスコード、JSONサンプルを記載
- 共通仕様（認証方式、日時形式、エラーレスポンス形式）をまとめて記載
- エンドポイント一覧テーブルを末尾に追加

### 変更ファイル一覧

- `docs/development/api-specification.md` - 新規作成
- `CLAUDE.md` - 開発ガイドセクションにAPI仕様書リンクを追加
