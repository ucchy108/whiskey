# 運動強度スコア（daily_score）設計

ヒートマップの色の濃さを決定する日別の運動強度スコアの設計ドキュメント。

## 概要

各ワークアウトに対して 0〜100 の `daily_score` を計算し、ヒートマップで「どれくらい頑張ったか」を可視化する。

## 計算ロジック

### バックエンド: `CalculateDailyScore`

**ファイル**: `backend/domain/entity/workout.go`

```
スコア = セット数 × 5
  + (総ボリューム > 1,000kg → +10)
  + (総ボリューム > 5,000kg → +10)
  + (総ボリューム > 10,000kg → +10)
  上限: 100
```

- **セット数**: ワークアウト内の全セット数
- **総ボリューム**: 各セットの `重量(kg) × レップ数` の合計

### 計算例

| ワークアウト | セット数 | 総ボリューム | スコア |
|---|---|---|---|
| ベンチプレス 60kg×10 の1セット | 1 | 600kg | 5 |
| ベンチプレス 80kg×8 の3セット | 3 | 1,920kg | 25 |
| スクワット 100kg×5 の5セット + デッドリフト 120kg×3 の3セット | 8 | 3,580kg | 50 |
| フルメニュー 15セット、総ボリューム 6,000kg | 15 | 6,000kg | 95 |
| 高強度 20セット、総ボリューム 15,000kg | 20 | 15,000kg | 100 |

### 呼び出しタイミング

`recalculateDailyScore()` が以下のタイミングで自動実行される:

- ワークアウト記録時（`RecordWorkout`）
- セット追加時（`AddWorkoutSets`）
- セット削除時（`DeleteWorkoutSet`）

**ファイル**: `backend/usecase/workout_usecase.go`

## フロントエンド: スコア → ヒートマップレベル変換

**ファイル**: `frontend/src/features/dashboard/hooks/useHeatmapGrid/useHeatmapGrid.ts`

`daily_score`（0〜100）をヒートマップの5段階レベルに変換する。

| スコア範囲 | レベル | 色 |
|---|---|---|
| 0 | 0 | 灰（`heatmap.level0`） |
| 1〜20 | 1 | 薄緑（`heatmap.level1`） |
| 21〜45 | 2 | 緑（`heatmap.level2`） |
| 46〜70 | 3 | 濃緑（`heatmap.level3`） |
| 71〜100 | 4 | 最濃緑（`heatmap.level4`） |

## 既知の課題

現在のロジックはセット数が主要因子であり、重量・レップ数の影響が薄い。トレーニングボリューム（総負荷量）ベースへの変更を予定している。

詳細: [GitHub Issue #36](https://github.com/ucchy108/whiskey/issues/36)
